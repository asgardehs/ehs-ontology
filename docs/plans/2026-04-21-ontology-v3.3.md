# Ontology v3.3 + Mimir viewer — Plan (2026-04-21)

Scope: ontology-side work supporting odin's Phase 4 (OSHA ITA CSV
export). Extracted from odin's
`docs/plans/2026-04-21-osha-ita-csv-export.md` so that edits to the
ontology or its tooling live in one repo, not two.

Odin bumps its submodule pointer at `third_party/ehs-ontology/` once
v3.3 lands on this repo's `main`.

---

## Principles

- **Ontology is the source of truth.** ITA is a reporting surface in
  odin, not a new domain — but the fields odin needs (EIN,
  establishment size and type, 4th narrative, ITA outcome + case-type
  taxonomies) belong here first. No SQL column lands in odin until
  the concept is in `ehs-ontology-v3.3.ttl` and HermiT validates clean.
- **Translation from Odin taxonomy to ITA taxonomy is declarative.**
  Odin's `severity_code` and `case_classification` codes do not match
  ITA enum codes 1-to-1. The mapping lives here as SKOS triples
  (`skos:exactMatch` / `skos:broadMatch`) and is mirrored into odin's
  SQL via seeded mapping tables — never hardcoded in Go.

## Phase 1 — v3.3 TTL edits

Extend `ehs-ontology-v3.2.ttl` → `ehs-ontology-v3.3.ttl`.

### New classes

- `ehs:EmployerIdentificationNumber` — literal data property on
  `ehs:Establishment`.
- `ehs:EstablishmentSize` — SKOS concept scheme with three concepts:
  - `ehs:EstablishmentSize_Small` (≤ 19 employees)
  - `ehs:EstablishmentSize_Medium` (20–249)
  - `ehs:EstablishmentSize_Large` (≥ 250)
  Thresholds from OSHA 29 CFR 1904.41.
- `ehs:EstablishmentType` — SKOS concept scheme with three concepts:
  - `ehs:EstablishmentType_PrivateIndustry`
  - `ehs:EstablishmentType_StateGovernment`
  - `ehs:EstablishmentType_LocalGovernment`
- `ehs:TreatmentFacilityType` — SKOS concept scheme (Hospital ER /
  Hospital outpatient clinic / Physician's office / Urgent care /
  Occupational health clinic / Other / Unknown).
- `ehs:ITAIncidentOutcome` — SKOS concept scheme (Death / Days Away /
  Job Transfer or Restriction / Other Recordable).
- `ehs:ITAIncidentType` — SKOS concept scheme (Injury / Skin Disorder /
  Respiratory Condition / Poisoning / Hearing Loss / Other Illness).

### New object and data properties

- `ehs:hasEIN rdfs:range xsd:string`
- `ehs:hasEstablishmentSize rdfs:range ehs:EstablishmentSize`
- `ehs:hasEstablishmentType rdfs:range ehs:EstablishmentType`
- `ehs:hasCompanyName rdfs:range xsd:string`
- `ehs:hasDaysAwayFromWork rdfs:range xsd:integer`
- `ehs:hasDaysRestrictedOrTransferred rdfs:range xsd:integer`
- `ehs:hasDateOfDeath rdfs:range xsd:date`
- `ehs:hasTreatmentFacilityType rdfs:range ehs:TreatmentFacilityType`
- `ehs:hasTimeUnknown rdfs:range xsd:boolean`
- `ehs:hasInjuryIllnessDescription rdfs:range xsd:string`
- `ehs:hasITAOutcome rdfs:range ehs:ITAIncidentOutcome`
- `ehs:hasITAType rdfs:range ehs:ITAIncidentType`

### SKOS mappings (Odin taxonomy → ITA)

- Every existing `ehs:SeverityLevel_*` individual gets either
  `skos:exactMatch` or `skos:broadMatch` to one
  `ehs:ITAIncidentOutcome_*`.
- Every existing `ehs:CaseClassification_*` gets
  `skos:exactMatch ehs:ITAIncidentType_*`.

### Deliverable

- `ehs-ontology-v3.3.ttl` (v3.2 copy + additions).
- `CHANGELOG.md` entry for v3.3.
- `README.md` pointer updated.
- `tests/run.sh ehs-ontology-v3.3.ttl` passes — all 8 existing
  scenario ASK queries plus one new `scenario-9-ita-export.rq` for
  the ITA routing added in v3.3.
- HermiT validation clean (0 unsat classes, 0 warnings beyond the
  pre-existing `xsd:duration` note).
- Merged to `main` so odin can bump its submodule pointer.

## Phase 2 — Mimir TURTLE graph viewer (optional, after Phase 1)

Browser-based viewer for `ehs-ontology-v3.3.ttl` and prior versions.
Motivated by v3.3's sprawl: five SKOS concept schemes plus mapping
triples on top of v3.2's eight modules. Reading raw TTL to verify
SKOS links (e.g. does every `ehs:SeverityLevel_*` individual
actually resolve to an `ehs:ITAIncidentOutcome_*`?) is slow and
error-prone. A visual walker over the graph pays for itself the
first time an ontology regression slips through HermiT.

### Repo layout (locked)

- New sibling repo `asgardehs/mimir` — viewer code lives there.
- This repo (`ehs-ontology`) adds `tools/mimir/` as a git submodule
  pointing at `asgardehs/mimir` main, so
  `git clone --recurse-submodules` of the ontology repo pulls the
  viewer alongside the TTL it inspects.

### Stack (locked)

- **Python + rdflib** — parsing and SPARQL execution.
- **FastAPI** — thin JSON API over rdflib. Chosen over Django: the
  viewer is a JS-heavy SPA with no use for Django's ORM / admin /
  templates / auth, and FastAPI's auto-generated OpenAPI docs at
  `/docs` double as free SPARQL-endpoint documentation. Revisit only
  if Mimir grows into a multi-user ontology management surface with
  persisted annotations or edit workflows.
- **Cytoscape.js** — frontend graph rendering. Client-side layout
  handles v3.3's triple count (~low thousands).
- **SPARQL sandbox** — textarea POSTs to a FastAPI endpoint, rdflib
  executes, JSON back. Pre-load the scenario queries from this
  repo's `tests/scenario-*.rq` as selectable presets.

### MVP features

- Load a TTL file (drop-down of versions under the submodule-mounted
  ontology repo, or upload).
- Class browser: tree view rooted at `owl:Thing`, expandable by
  `rdfs:subClassOf`.
- Node detail panel: labels, comments, properties (data + object),
  `skos:exactMatch` / `skos:broadMatch` links rendered as clickable
  cross-references.
- SKOS concept-scheme view: one panel per `skos:ConceptScheme` with
  all its concepts and their external mappings.
- SPARQL sandbox.
- Diff view (stretch): load v3.2 and v3.3 side-by-side, highlight
  added/removed triples.

### Out of scope for MVP

- Editing (viewer only — edits stay in Zed + CLI).
- Reasoning (delegate to HermiT via `tests/run.sh`).
- Multi-ontology federation.
- Auth / multi-user.

### Deliverable

In `asgardehs/mimir`:
- `README.md` — setup, supported TTL versions, screenshots.
- `pyproject.toml` (uv or poetry, TBD at build time).
- Docker compose for one-command local spin-up.
- Smoke test: load v3.3, assert every `ehs:SeverityLevel_*`
  individual reaches an `ehs:ITAIncidentOutcome_*` via SKOS.

In this repo (separate commit after Mimir's first release):
- `tools/mimir/` submodule added, pointing at `asgardehs/mimir` main.
- `README.md` gets a short "Tools" section describing Mimir and the
  submodule command to pull it.

### Sequencing

Phase 2 is **nice-to-have**, not a prerequisite. The v3.3 edit path
(Zed + CLI + HermiT + `tests/run.sh`) already works end-to-end.
Build after v3.3 ships — Mimir's value is highest once there's a
non-trivial SKOS mapping surface to inspect.
